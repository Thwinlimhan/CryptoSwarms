- What the v1–v5 research actually found (vs what sounded good in theory)

## 1. EFFICIENCY PRINCIPLES

These are not aspirational. Every architectural decision below must pass all five.

### Principle 1 — Prove the Loop Before Adding Agents

One signal source → one backtest engine → one exchange → one strategy type.
Get that pipeline generating paper-traded P&L before adding a second agent anywhere.
**Violation signal:** You’re writing a new agent before the previous one has produced a backtest result.

### Principle 2 — Don’t Build What Exists

Three things you were going to build from scratch that now don’t need building:

- Mission Control dashboard (builderz-labs built it, MIT license, 28 panels)
- Memory extraction logic (Mem0 does this in 3 lines, Netflix uses it in production)
- Stealth browsing (Camoufox patches Firefox at C++ level — you can’t build this)

**Violation signal:** You write >50 lines of code for something with a maintained open-source equivalent.

### Principle 3 — Fail Fast on Strategies, Not on Infrastructure

The 5-gate validation funnel exists to kill bad strategies before they touch capital.
Most strategies will fail Gate 2 (fast screen). This is correct behavior.
The system should make running a new backtest take <5 minutes of human effort.
**Violation signal:** A new strategy hypothesis takes >1 day to get to backtest results.

### Principle 4 — Local LLM First, Always

Qwen3.5 4B on SGLang should handle 80%+ of inference.
If a task “needs” Claude Sonnet, ask whether it actually needs it or whether you’re being lazy about prompting.
Set a hard $10/day LLM budget enforced by Paperclip from Day 1.
**Violation signal:** Daily LLM costs consistently exceed $5 before you have live strategies.

### Principle 5 — Evolution Swarm Is Month 8+

The genetic optimizer needs real P&L data to produce meaningful signal.
Running DEAP on 2 strategies with 30 days of paper trades produces noise, not insight.
Build the infrastructure in Phase 4, activate it seriously in month 8–12.
**Violation signal:** You’re tuning the genetic optimizer parameters before you have 10+ live strategies.

### Signal Flow — Every Step

```
STEP 1  DETECT
        Market Scanner runs every 15 min
        → Publishes signal to redis:research:signals
        → Mem0.add(signal_summary, user_id="market_scanner")

STEP 2  QUALIFY
        Signal Aggregator scores sources by reliability
        → Confidence >= 0.65 AND signal_age < 30min → enter backtest queue
        → Below threshold → log and discard

STEP 3  CODE
        Strategy Coder translates hypothesis → Jesse code
        → Pine Script MCP validates logic (not syntax — logic)
        → First draft: Qwen3.5 4B local
        → Final review: GLM-5

STEP 4  VALIDATE (6 gates, fail-fast ordered)
        Gate 1 Syntax          → reject malformed code immediately
        Gate 2 Sensitivity     → ±10% param perturbation, Sharpe must stay >0.7
        Gate 3 Fast Screen     → VectorBT 1000-combo sweep, dual slippage
        Gate 4 Walk-Forward    → Jesse WFE >= 0.6 across 4 folds
        Gate 5 Regime Test     → profitable in >= 2 of 4 regimes
        Gate 6 Correlation     → Pearson < 0.4 with any existing strategy
        → All pass → enter 30-day paper trading

STEP 5  OBSERVE
        Paper trading for 30 days minimum
        → Performance Analyst logs nightly via Mission Control
        → Human reviews weekly in Mission Control dashboard

STEP 6  PROMOTE
        Human types CONFIRM in Mission Control or CoPaw
        → Allocation: $500–$1000 initial (never more than this on first live)

STEP 7  EXECUTE
        Smart Router selects exchange (size, MEV risk, asset type)
        → Paperclip checks budget cap
        → Dead Man's Switch confirms Risk Monitor heartbeat
        → CONFIRM gate (user types CONFIRM via CoPaw)
        → Order via exchange skill
        → Log to TimescaleDB + Mission Control

STEP 8  EVOLVE (Phase 4+)
        Nightly DEAP: mutate parameters of live strategies
        → Graphiti: update strategy-regime knowledge graph
        → Mem0: extract new performance facts automatically
        → 90-day Sharpe < 0.5 → auto-retire, archive in Neo4j
```

## 6. MEMORY ARCHITECTURE

### Layer Assignments

```
Mem0 + Qdrant       → All research agents (fast working memory)
Graphiti + Neo4j    → Evolution + Backtest (temporal knowledge graph)
Letta               → Evolution Swarm only, Phase 4+ (agent autonomy)
TimescaleDB         → Quantitative data (never LLM-managed — raw numbers)
```

### Unified Memory Client

```python
# memory/agent_memory.py
from mem0 import Memory
from graphiti_core import Graphiti
import asyncio, time

class AgentMemory:
    """
    Single interface for all agent memory operations.
    Agents call .remember() and .recall() — never touch Mem0/Graphiti APIs directly.
    """

    def __init__(self, agent_name: str):
        self.agent_id = agent_name
        self.fast = Memory(config={
            "vector_store": {
                "provider": "qdrant",
                "config": {"url": QDRANT_URL, "collection_name": "swarm_memory"}
            },
            "llm": {
                "provider": "openai",
                "config": {"base_url": SGLANG_URL, "model": "qwen3.5-4b"}
            }
        })
        self.graph = Graphiti(NEO4J_URI, NEO4J_USER, NEO4J_PASS)

    def remember(self, text: str, important: bool = False):
        """Store in fast memory. Promote to temporal graph if important."""
        self.fast.add(text, user_id=self.agent_id)
        if important:
            asyncio.run(self.graph.add_episode(
                name=f"{self.agent_id}_{int(time.time())}",
                episode_body=text,
                source_description=f"Agent: {self.agent_id}",
            ))

    def recall(self, query: str, limit: int = 5) -> list[str]:
        """Fast semantic search in working memory."""
        results = self.fast.search(query, user_id=self.agent_id, limit=limit)
        return [r["memory"] for r in results["results"]]

    def recall_history(self, query: str) -> list:
        """Temporal graph query — how did this knowledge evolve?"""
        return asyncio.run(self.graph.search(query, num_results=10))

    def forget(self, query: str):
        """Remove outdated facts (e.g. strategy retired)."""
        results = self.fast.search(query, user_id=self.agent_id)
        for r in results["results"]:
            self.fast.delete(memory_id=r["id"])
```

```python
# DeerFlow task prompt template
ALPHA_MINING_PROMPT = """
You are an alpha researcher for a systematic crypto trading system.

SIGNAL RECEIVED:
{signal_json}

CURRENT MARKET REGIME: {regime}

YOUR MISSION — complete these steps in order:

STEP 1: MEMORY CHECK
  Search your memory for similar past signals:
  Query: "{symbol} {signal_type} historical outcome"
  What happened after this signal type historically?
  Did it work or fail? Under what regime?

STEP 2: HISTORICAL CONTEXT SEARCH
  Search: "{symbol} {signal_type} historical pattern crypto"
  Search: "when BTC funding rate was {funding_value} what happened next"
  Find 3 specific historical analogs with dates and outcomes.
  Use real data. If you can't find 3 analogs, say so — don't fabricate.

STEP 3: HYPOTHESIS GENERATION
  Generate 2 hypotheses (primary + alternative). For each:
  - name: descriptive slug (snake_case)
  - type: trend_follow | mean_reversion | momentum | stat_arb
  - entry_condition: mathematical expression (precise)
  - stop_loss: specific price rule (not "use common sense")
  - take_profit: specific R multiple or price rule
  - timeframe: 1h | 4h | 1d
  - regime_fit: which regimes does this work in?
  - edge_source: WHY should this work? Cite specific evidence.

STEP 4: PRIORITY SCORE (0–100 each hypothesis)
  Score rubric:
    Historical win rate evidence  0–30 pts
    Current regime fit            0–20 pts
    Implementation simplicity     0–20 pts  (less code = less bugs)
    R:R estimate                  0–20 pts
    Portfolio novelty             0–10 pts  (different from existing strategies)

STEP 5: JESSE CODE for the top-scored hypothesis
  Write complete Jesse strategy class. Rules:
  - Use ONLY ta.rsi(), ta.ema(), ta.adx(), ta.bb() from jesse.indicators
  - NEVER implement indicators from scratch
  - ALWAYS use risk_to_qty(2.0, stop_loss) for position sizing
  - ALWAYS define stop_loss property before go_long/go_short
  - Include hyperparameters() method for later Hyperopt
  - Add inline comment for every non-obvious line

OUTPUT JSON:
{
  "hypotheses": [
    {
      "name": str,
      "type": str,
      "priority_score": int,
      "jesse_code": str,
      "regime_fit": [str],
      "expected_sharpe_range": [float, float],
      "key_risk": str,
      "evidence_sources": [str]
    }
  ],
  "recommended_for_backtest": str,
  "insufficient_evidence": bool   // true if you couldn't find 3 real analogs
}

HARD RULES:
  If insufficient_evidence is true → output the flag and stop. Do not fabricate analogs.
  If priority_score < 50 on best hypothesis → flag as LOW_CONFIDENCE, do not queue.
  Jesse code must compile — validate logic with pine-script MCP before submitting.
"""
```

```python
# Agent implementation
from deerflow import DeerFlowClient

class AlphaMiningAgent:
    def __init__(self):
        self.client = DeerFlowClient(
            model="claude-sonnet-4-6",
            memory_backend="local",        # NO TIAMAT cloud memory
        )
        self.memory = AgentMemory("alpha_mining")

    def process_signal(self, signal: dict) -> None:
        regime = self.redis.get("regime:current")

        result = self.client.run(
            task=ALPHA_MINING_PROMPT.format(
                signal_json=json.dumps(signal, indent=2),
                regime=regime,
                symbol=signal["symbol"],
                signal_type=signal["signal_type"],
                funding_value=signal.get("data", {}).get("funding_rate", "N/A"),
            ),
            skills=["crypto-research", "pine-script"],
            max_steps=25,
            timeout_minutes=15,
        )

        data = json.loads(result.output)

        if data.get("insufficient_evidence"):
            self.memory.remember(f"Insufficient evidence for {signal['symbol']} {signal['signal_type']}")
            return

        best = max(data["hypotheses"], key=lambda h: h["priority_score"])

        if best["priority_score"] >= 50:
            self.redis.xadd("backtest:queue", {
                "source": "alpha_mining",
                "hypothesis": json.dumps(best),
                "signal": json.dumps(signal),
                "queued_at": str(time.time()),
            })
            self.memory.remember(
                f"Queued hypothesis '{best['name']}' (score {best['priority_score']}) "
                f"for {signal['symbol']}",
                important=True
            )
```

## 10. BACKTEST SWARM

### The 6-Gate Pipeline (with efficiency ordering)

Fail-fast: cheapest gates run first. Expensive computation only when cheap gates pass.

```
Gate 1 — SYNTAX (< 5s)
  Does the Jesse code import correctly? Does it compile?
  If no: return to Strategy Coder with error message

Gate 2 — SENSITIVITY (< 2 min)
  Perturb each parameter ±10%. Run fast backtest on each.
  Sharpe must stay > 0.7 across all perturbations.
  Purpose: catch overfit strategies before expensive computation.
  If fail: REJECT. Don't waste walk-forward compute on fragile params.

Gate 3 — FAST SCREEN, DUAL SLIPPAGE (< 3 min)
  VectorBT: 1000-param sweep, 1-year data.
  Run at BOTH slippage scenarios:
    Optimistic:    fee=0.001, slippage=0.001
    Conservative:  fee=0.001, slippage=0.004
  PASS criteria (must meet at conservative scenario):
    Sharpe > 0.7, expectancy > 0, max_dd < 30%, win_rate > 35%
  If only passes optimistic: REJECT with note "slippage-sensitive"

Gate 4 — WALK-FORWARD (5-10 min)
  Jesse walk-forward: 4 folds, 9-month IS / 3-month OOS
  PASS: WFE ratio > 0.6 on ALL 4 folds
  WFE interpretation: 0.8+ excellent, 0.6-0.8 good, <0.6 overfit

Gate 5 — REGIME TESTING (5-10 min)
  Run strategy isolated in each regime period:
    trending_up, trending_down, mean_reverting, high_volatility
  PASS: profitable (Sharpe > 0.3) in >= 2 of 4 regimes
  Tag result: "bull_only", "bear_resistant", "all_weather", etc.
  This tag determines future regime-based allocation weight.

Gate 6 — CORRELATION CHECK (< 1 min)
  Fetch active strategy daily returns from TimescaleDB.
  Pearson correlation vs all active strategies.
  PASS: < 0.4 with any single strategy, < 0.5 with equal-weighted portfolio
  If fail: note as REDUNDANT — different params won't fix this.

→ All 6 gates pass: enter 30-day paper trading
→ Any gate fails: ARCHIVE with full gate-by-gate result stored in TimescaleDB
  (archived strategies are training data for Evolution Swarm later)

### Data Quality Check (runs before every backtest)

```python
# backtesting/data_quality.py
def validate_candles(candles: np.ndarray, timeframe_h: int) -> DataQualityReport:
    """
    Validate candle data before running any backtest.
    Garbage in = garbage backtest results.
    """
    n = len(candles)
    coverage_days = (n * timeframe_h) / 24

    issues = []
    if coverage_days < 365:
        issues.append(f"Insufficient history: {coverage_days:.0f} days (need 365)")

    # Gap detection (missing candles)
    timestamps = candles[:, 0]
    expected_gap = timeframe_h * 3600
    actual_gaps = np.diff(timestamps)
    gap_count = np.sum(actual_gaps > expected_gap * 1.5)
    gap_pct = gap_count / n
    if gap_pct > 0.005:
        issues.append(f"Too many gaps: {gap_pct:.1%} (max 0.5%)")

    # Outlier detection (price spikes)
    log_returns = np.diff(np.log(candles[:, 4]))  # close prices
    z_scores = np.abs((log_returns - log_returns.mean()) / log_returns.std())
    outlier_count = np.sum(z_scores > 5)
    if outlier_count / n > 0.001:
        issues.append(f"Outliers detected: {outlier_count} candles with |z| > 5")

    # Zero volume candles
    zero_vol = np.sum(candles[:, 5] == 0) / n
    if zero_vol > 0.01:
        issues.append(f"Zero volume candles: {zero_vol:.1%}")

    if issues:
        raise DataQualityError(issues)
    return DataQualityReport(ok=True, coverage_days=coverage_days)
```

    "ema_fast": 20, "ema_slow": 50,
    "rsi_period": 14, "rsi_threshold": 30,
    "risk_pct": 2.0, "tp_ratio": 2.0,

class {StrategyName}(Strategy):

    @property

                self.ema_fast > self.ema_slow)

    def should_cancel_entry(self) -> bool: return False

        risk = abs(self.average_entry_price - self.stop_loss)
        return self.average_entry_price + (risk * CONFIG["tp_ratio"])

### Genetic Optimizer (DEAP) — Phase 4+

```python
# evolution/genetic_optimizer.py
from deap import base, creator, tools, algorithms

# Fitness: maximize Sharpe × Calmar, penalize drawdown
creator.create("FitnessMax", base.Fitness, weights=(1.0,))
creator.create("Individual", list, fitness=creator.FitnessMax)

class GeneticOptimizer:
    POPULATION_SIZE = 200
    GENERATIONS     = 50
    CROSSOVER_PROB  = 0.7
    MUTATION_PROB   = 0.2

    def _fitness(self, individual: list) -> tuple:
        params = self._decode(individual)
        result = run_jesse_backtest(self.strategy, params,
                                    slippage=0.004,    # Always conservative
                                    quick=True)
        score = (result.sharpe * result.calmar) / (1 + abs(result.max_drawdown))
        return (score,)

    def evolve(self) -> list:
        toolbox = self._build_toolbox()
        population = toolbox.population(n=self.POPULATION_SIZE)
        final_pop, _ = algorithms.eaSimple(
            population, toolbox,
            cxpb=self.CROSSOVER_PROB, mutpb=self.MUTATION_PROB,
            ngen=self.GENERATIONS, verbose=False,
        )
        return [self._decode(ind) for ind in tools.selBest(final_pop, k=5)]

    # Efficiency note: run on conservative slippage ALWAYS
    # Don't evolve parameters that only work at optimistic conditions
